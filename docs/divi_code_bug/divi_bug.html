Divi Code Bug

1. E-Mail Customizer Divi Bug

Wie behebe ich diesen Fehler?

Die Datei order-items-rows-4.php muss unter folgendem Pfad per FTP-Zugang verändert werden:
/wp-content/plugins/email-customizer-for-woocommerce-with-drag-drop-builder/templates/ec-woo-mail-helper/order-items-rows-4.php

Hierbei muss folgende Codezeile gelöscht oder auskommentiert werden:
do_action('woocommerce_order_item_meta_end', $item_id, $item, $order, $plain_text);
'woocommerce_order_item_meta_end' enthält die gesamte Description, die in diesem Fall aus dem Divi Code besteht.

Nachdem diese Codezeile gelöscht wurde, kann man seinen eigenen Code erstellen.
wc_get_product holt alle Infos von dem jeweiligen item: $product = wc_get_product( $item->get_product_id() );
get_short_description holt die kurze Beschreibung des jeweiligen Produkts:

$product_short_description = $product->get_short_description();

Damit dies in der E-Mail angezeigt wird, muss folgender Code noch eingefügt werden:
echo '<div class="bsp_container">' .
  '<p>' . $product_short_description . '</p>'
.'</div>';

2. WooCommerce Code Bug in Bestelldetails

Wie behebe ich diesen Fehler?

Um den Code von der Seite /kasse/order-received
zu bekommen muss die Datei order-details-item.php im WooCommerce Plugin werden angepasst unter:
/wp-content/plugins/woocommerce/templates/order/order-details-item.php

Die Zeile 50 muss auskommentiert werden.:
do_action( 'woocommerce_order_item_meta_end', $item_id, $item, $order, false );
